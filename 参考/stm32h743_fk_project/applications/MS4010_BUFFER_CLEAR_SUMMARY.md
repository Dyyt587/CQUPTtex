# MS4010驱动缓冲区清空功能完成

## 改进概述

✅ **已完成**: 为MS4010驱动添加了RS485缓冲区清空功能，当出现接收错误时自动清空串口设备缓冲区，防止后续控制命令帧接收解析错误。

## 主要改进

### 1. 新增缓冲区清空函数
- **函数名**: `ms4010_clear_rx_buffer()`
- **功能**: 智能清空RS485接收缓冲区
- **策略**: 中断接收 → 循环读取 → 清空残留数据

### 2. 错误处理增强
- **接收超时错误**: 自动清空缓冲区
- **帧格式错误**: 自动清空缓冲区  
- **解析失败错误**: 自动清空缓冲区
- **双模式支持**: 单电机和多电机模式均适用

### 3. 实现细节
```c
// 核心功能
static rt_err_t ms4010_clear_rx_buffer(rs485_inst_t *rs485_handle)
{
    /* 中断当前接收 */
    rs485_break_recv(rs485_handle);
    
    /* 循环清空缓冲区 */
    while (clear_count < max_clear_attempts) {
        recv_len = rs485_recv(rs485_handle, dummy_buffer, sizeof(dummy_buffer));
        if (recv_len <= 0) break;  // 缓冲区已清空
    }
    
    /* 恢复超时设置 */
    rs485_set_recv_tmo(rs485_handle, MS4010_DEFAULT_TIMEOUT);
}
```

## 错误处理触发点

### ms4010_send_command函数
1. **接收长度不符** → 清空缓冲区
2. **协议解析失败** → 清空缓冲区

### ms4010_send_command_shared函数  
1. **接收长度不符** → 清空缓冲区
2. **响应帧头错误** → 清空缓冲区
