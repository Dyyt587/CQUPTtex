# Home点坐标转换修改说明

## 问题描述

用户发现当设置坐标为(0,0)时，俯仰轴电机角度为90°，但期望这时应该是0°。也就是说，xy坐标的原点(0,0)应该对应home点，此时激光应该水平指向正前方。

## 原始问题分析

### 原始转换逻辑
```c
float horizontal_distance = sqrtf(x*x + y*y);
angle->yaw = atan2f(y, x);  // 当x=y=0时，结果未定义
angle->pitch = atan2f(-height_diff, horizontal_distance);  // 当horizontal_distance=0时，结果为±90°
```

### 问题原因
1. **偏航角计算**: `atan2(0, 0)` 在数学上未定义，不同实现可能返回不同值
2. **俯仰角计算**: 当`horizontal_distance = 0`时，`atan2(-height_diff, 0)` 会返回±90°，而不是期望的0°

## 修改方案

### 新的转换逻辑
```c
/* 计算水平距离 */
float horizontal_distance = sqrtf(x*x + y*y);

/* 处理home点情况 */
if (horizontal_distance < 1e-6f) // 接近原点时
{
    angle->yaw = 0.0f;    // home点，偏航角为0
    angle->pitch = 0.0f;  // home点，俯仰角为0 (水平指向前方)
}
else
{
    // 正常的角度计算逻辑
    angle->yaw = LASER_GIMBAL_RAD_TO_DEG(atan2f(y, x));
    // ... 正常的俯仰角计算
}
```

### 修改要点

1. **阈值判断**: 使用`1e-6f`作为判断阈值，避免浮点数精度问题
2. **Home点定义**: 当(x,y)接近(0,0)时，强制设置为home点
3. **角度设置**: Home点的偏航角和俯仰角都设为0°
4. **Z坐标忽略**: 在home点情况下，Z坐标不影响角度计算

## 坐标系统定义

### Home点含义
- **坐标**: (0, 0, z) - z可以是任意值
- **偏航角**: 0° (正前方)
- **俯仰角**: 0° (水平)
- **物理含义**: 激光水平指向正前方

### 坐标系统
```
       Y (向右)
       ↑
       |
       |
Z ←----+----→ X (向前)
       |
       |
       ↓
   (向左)
```

### 角度定义
- **偏航角 (Yaw)**:
  - 0° = 正前方 (X轴正方向)
  - +90° = 向右 (Y轴正方向)  
  - -90° = 向左 (Y轴负方向)
  - ±180° = 正后方 (X轴负方向)

- **俯仰角 (Pitch)**:
  - 0° = 水平 (Home点)
  - +角度 = 向下
  - -角度 = 向上

## 测试验证

### 新增测试命令
```bash
laser_home_test
```

此命令会：
1. 测试坐标(0,0,1)的角度转换
2. 验证结果是否为(0°,0°)
3. 实际控制激光指向home点
4. 输出详细的测试结果

### 预期输出
```
=== Home Point Test ===
✅ Home coordinate (0,0,1.0) -> Angle (0.00°, 0.00°)
Expected: Yaw=0°, Pitch=0° (should match!)

=== Test completed ===
```

## 使用示例

### 基本控制
```bash
# 回到home点 (方法1: 直接调用home函数)
laser_home

# 回到home点 (方法2: 使用坐标控制)
laser_point 0 0 1.0

# 回到home点 (方法3: 使用角度控制)
laser_angle 0 0
```

### 校准时的home点
```bash
# 进入校准模式
laser_calibration on

# 手动调整到home位置
laser_cal_set 0 0

# 告诉系统这是home点
laser_cal_offset 0 0

# 保存校准
laser_cal_save

# 退出校准模式
laser_calibration off
```

## 优势效果

### 1. 直观性提升
- 坐标(0,0)直接对应home点
- 不需要记忆复杂的角度值
- 符合直觉的操作方式

### 2. 数值稳定性
- 避免了atan2(0,0)的未定义行为
- 消除了除零错误的可能性
- 提供确定性的结果

### 3. 一致性保证
- 无论Z坐标是什么值，(0,0,z)都指向home点
- 与现有的home函数行为一致
- 与校准系统的0度基准一致

## 兼容性说明

### 向后兼容
- 所有现有API保持不变
- 原有的角度控制功能不受影响
- 校准参数格式保持一致

### 行为变化
- **变化前**: 坐标(0,0,z)可能产生不确定的角度
- **变化后**: 坐标(0,0,z)始终产生(0°,0°)角度
- **影响**: 只影响原点附近的坐标，其他坐标计算不变

## 注意事项

1. **阈值敏感性**: 使用1e-6作为阈值，足够小以避免误判
2. **Z坐标处理**: Home点时Z坐标被忽略，这是设计行为
3. **校准重要性**: 确保0°角度确实对应物理上的home位置
4. **测试建议**: 使用`laser_home_test`命令验证修改效果
